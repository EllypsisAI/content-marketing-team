## 1. Text Post Craft

### Character Limits & Sweet Spots

| Element | Limit | Sweet spot |
|---------|-------|------------|
| Regular post | 3,000 chars | 1,300–1,900 chars (47% higher engagement) |
| "See more" cutoff | ~210–235 chars visible | First 2 lines = the entire decision |
| Comment | 1,250 chars | — |
| Connection request note | 300 chars | — |

Posts under 500 chars: algorithm reads as low-effort. Posts over 2,000 chars: reader drop-off before engagement.

### Post Structure (recommended arc)

| Section | Chars | Purpose |
|---------|-------|---------|
| Hook | ~110–210 | Claim, tension, or story opening. Must survive the "see more" cut |
| Context | 300–400 | Set up the problem or situation |
| Insight | 500–700 | Core value delivery — the thing they came for |
| CTA | 100–200 | Question, invitation to comment, or reframe |

Total: ~1,300–1,500 chars. Add a personal aside or example to reach the 1,900 ceiling when warranted.

### Hook Patterns (ranked by fit for Ellypsis)

| Pattern | Template | When to use | Example |
|---------|----------|-------------|---------|
| **Contrarian claim** | "[Common belief] is wrong. Here's what [data/experience] shows." | Pillar 2 (Augmentation), challenging hype | "Most AI projects fail at the translation layer, not the technology." |
| **Story drop-in** | "[Specific scene]. [Unexpected detail]." | Pillar 1 (Implementation), client stories | "I sat with a woman who was opening 10 windows to answer a single email about rust on a ship hull." |
| **Specific result** | "[Metric] in [timeframe]. Here's what we actually did." | Pillar 1, proof points | "We cut a 45-minute sales research process to 8 minutes. Here's the before/after." |
| **CEO scenario** | "Din CEO har laest en artikel om [X]. Nu skal I have [X]. Her er hvad du goer." | Pillar 2, Danish-language | Proven format from Dennis Uhrskov Raagaard's engagement data |
| **Tension question** | "Denmark leads Europe in AI adoption. So why are 43% of SMEs still doing nothing?" | Pillar 3, landscape | Data tension creates cognitive open loop |
| **Bold reframe** | "The question isn't [common frame]. It's [better frame]." | Pillar 2, philosophy | "The question isn't 'what becomes unnecessary?' It's 'what becomes possible?'" |
| **Benefit promise** | "This [X]-minute framework will change how you [specific action]." | Pillar 2, frameworks | Use sparingly — can feel clickbaity |

**Hook quality test:** Read only the first 2 lines. If they explain context → weak. If they make a claim worth reading → strong.

### Formatting Rules

| Rule | Why |
|------|-----|
| One sentence per line (or short paragraphs of 1–2 sentences) | Mobile readability. 57%+ of LinkedIn traffic is mobile |
| Double line break between sections | Creates visual breathing room, prevents wall-of-text |
| Bold sparingly (5–10% of content max) | Draws eye to key claims. Overuse kills the effect |
| Use → ✓ • for lists, not bullet points | Unicode symbols render more reliably than markdown |
| No more than 3 hashtags | 5+ hashtags correlates with 68% reach reduction |
| Hashtags at the end, not inline | Cleaner read. Hashtags function as metadata, not content |
| Never start with "I'm excited to announce" | Instant scroll-past. The feed is full of this |

### Hashtag Strategy

3 hashtags per post (LinkedIn's own recommendation):
- 1 broad: `#AI` or `#DigitalTransformation`
- 1 mid-range: `#AIImplementation` or `#DanskErhvervsliv`
- 1 niche: `#AugmentationOverAutomation` (own it) or `#DanskAI`

Niche hashtags outperform broad ones for reaching the right audience.

## 2. Carousel / Document Posts

### Why carousels dominate

| Metric | Value |
|--------|-------|
| Engagement rate | 6.60% (highest of all formats) |
| vs text-only | 596% higher engagement |
| Dwell time | 2–3x more than single-image or short text |
| Algorithm signal | High dwell time + comments = strongest amplification combo |

### Technical Specs

| Spec | Value |
|------|-------|
| Format | PDF (best compatibility across devices) |
| Dimensions | 1080 x 1080 px (square) or 1080 x 1350 px (portrait, better on mobile) |
| Max file size | 100 MB (keep under 3 MB for performance) |
| Max pages | 300 (technically) |
| Optimal pages | 6–12 slides for best engagement (drop-off after 10) |
| Min font size | 28px (mobile legibility) |
| All slides must be | Same dimensions throughout |

### Carousel Structure

| Slide | Purpose |
|-------|---------|
| **Slide 1 (Cover)** | Bold hook. Title + one compelling line. Visual brand consistency. This is the "see more" equivalent for carousels |
| **Slides 2–3** | Problem setup or context. Why this matters to the reader |
| **Slides 4–8** | Core content. One idea per slide. Big text, minimal clutter |
| **Slide 9–10** | Summary or key takeaway |
| **Final slide** | CTA: "Follow for more" / question / "Save this for later" |

### Carousel Content Types (best fit for Ellypsis)

| Type | Pillar | Example |
|------|--------|---------|
| Before/after workflow | 1 | "How we redesigned a 50-person company's order process" |
| Named framework | 2 | "The Augmentation Assessment: 5 Questions" |
| Data breakdown | 3 | "The State of AI Adoption in Denmark — What the Numbers Mean" |
| Step-by-step guide | 2 | "5 Questions to Ask Before Starting an AI Project" |
| Lessons learned | 1 | "3 Things We Find in Every AI Audit" |

### Companion Text

Always post with 3–5 lines of text introduction:
- Preview what's inside
- Make a bold statement or ask a question
- The text + carousel combo maximizes both dwell time and comment signals

## 3. Polls

### Mechanics

| Spec | Value |
|------|-------|
| Question length | Max 140 chars |
| Option length | Max 30 chars each |
| Number of options | 2–4 |
| Duration options | 1 day, 3 days, 1 week, 2 weeks |
| Engagement rate | 4.40% (doubled since 2023) |
| Reach | 3–5x more than regular posts |

### Best Practices

| Rule | Detail |
|------|--------|
| Frequency | 1–2x per month max (preserve novelty) |
| Duration | 1–3 days for urgency, 1 week for wider reach |
| Options | Represent real strategic choices, not obvious answers |
| Include "Other → comment" | Drives comment engagement (the metric that matters) |
| Follow up | Post results + your analysis. Creates a 2-post series |
| Strategic use | Market sensing, audience segmentation, content research, conversation ignition |

### Poll Templates for Ellypsis

| Question | Options | Pillar | Purpose |
|----------|---------|--------|---------|
| "Hvad stopper jeres virksomhed fra at implementere AI?" | A) Ved ikke hvor vi starter / B) Proevet det, virkede ikke / C) Intet budget / D) Mangler kompetencer | 2 | Audience intelligence + validates content angles |
| "How do you measure ROI on AI projects?" | A) Time saved / B) Cost reduction / C) Revenue impact / D) We don't yet | 1 | Surface the measurement gap |
| "Your CEO wants AI. What's your first move?" | A) Hire a consultant / B) Start a pilot / C) Train the team / D) Build a strategy | 2 | Engagement + lead qualification |

## 4. LinkedIn Newsletter

### Mechanics

| Spec | Value |
|------|-------|
| Available to | Creator Mode profiles |
| Subscriber notification | Push notification + email to all subscribers on publish |
| Content format | Rich text, images, embeds |
| Optimal length | 1,500–2,000 words (long-form) or 300–500 words (short-form) |
| Optimal frequency | Biweekly (for Ellypsis — balances depth + consistency with solo capacity) |
| Open rate benchmark | 35–40% (strong); below 25% after 10 editions = problem signal |
| Subscriber growth | 0–100 months 1–3 (cold start); 100–300 with existing network leverage |

### Newsletter Structure

| Section | Length | Purpose |
|---------|--------|---------|
| Hook paragraph | 1 para | Compelling opening that earns the read |
| Setup | 1 para | Context: why this matters now |
| Framework/Insight | 3 paras | Core value. The thing they subscribed for |
| Personal story/example | 1 para | Humanizes, builds connection, shows practitioner credibility |
| CTA | 1 para | Question, invitation, forward prompt |

### Content Strategy

- 80% pure value (education, insights, frameworks) / 20% promotional max
- Each newsletter should spawn 3–5 LinkedIn post seeds (excerpts, pull quotes, data points)
- Newsletter = depth layer. LinkedIn posts = distribution layer. They feed each other
- Name the newsletter something specific and ownable (not "AI Insights" — too generic)

## 5. Algorithm Signals

### Ranking Hierarchy (2026)

| Signal | Weight | Detail |
|--------|--------|--------|
| **Comments** | Highest | Length, threads, and velocity all matter. 15x more valuable than likes |
| **Comment velocity** | Critical | First 90 minutes determine amplification. Algorithm shows to 2–5% of network first |
| **Conversation depth** | Very high | Multi-turn exchanges between 3+ participants = 5.2x amplification |
| **Dwell time** | Very high | 61+ seconds = 15.6% avg engagement. <3 seconds = 1.2%. Carousels and long text optimize for this |
| **Expert comments** | Very high | Comments from industry experts carry 7–9x algorithmic value vs random connections |
| **Saves** | High | Indicates lasting value. Growing in importance |
| **Shares/Reposts** | High | Signal content worth spreading |
| **Topic authority** | Compound | 60+ days of thematic consistency = up to 78% visibility increase |
| **Interest-graph matching** | High | Algorithm categorizes posts by topic, serves to interested users regardless of connection |
| **Reactions** | Moderate | Helpful but least weighted of all engagement types |

### Algorithm Penalties

| Behavior | Impact |
|----------|--------|
| Engagement pods | Detected 97% accuracy, account-wide penalty |
| Engagement bait ("What do you think?") | Actively devalued |
| 5+ hashtags | 68% reach reduction |
| Content recycling (identical reposts) | 84% less reach |
| External links in post body | ~30% reach reduction (put in first comment instead) |
| AI-generated low-effort content | Penalized (LinkedIn claims "near human-level understanding" of content quality) |
| Editing post within 10 minutes of publish | Can reset distribution |

### Actionable Algorithm Rules

| Rule | Why |
|------|-----|
| Reply to every comment within 2 hours | Drives comment threads → 5.2x amplification |
| Ask follow-up questions to commenters | Creates multi-turn exchanges the algorithm rewards |
| Post between 8–10 AM (target timezone) | Maximizes the critical first-90-minute window |
| Tuesday–Thursday for business content | Highest professional engagement days |
| Saturday optional for Danish landscape content | Lower competition, engaged weekend readers |
| Minimum 12-hour gap between posts | Multiple same-day posts cannibalize each other |
| Put links in first comment, not post body | Avoids ~30% reach penalty |
| Pin the best comment | Surfaces conversation quality for new viewers |

## 6. Format Selection Matrix

For the `draft-social` skill to recommend format based on content type:

| Content type | Best format | Why | Dwell time |
|-------------|-------------|-----|------------|
| Implementation story (before/after) | Carousel (PDF) | Visual workflow comparison, high dwell | Very high |
| Named framework (step-by-step) | Carousel (PDF) | One concept per slide, saveable | Very high |
| Contrarian take / hot opinion | Text post | Raw, personal, immediate. Drives comments | Medium |
| Client lesson / failure post-mortem | Text post | Intimate, story-driven. Danish language | Medium–High |
| Data breakdown (stats, charts) | Carousel (PDF) | Visual data is more digestible | Very high |
| Audience intelligence gathering | Poll + text | 3–5x reach, drives comments | Low (but reach compensates) |
| Deep dive / long analysis | Newsletter | 1,500–2,000 words, depth layer | Very high (off-platform) |
| CEO scenario / relatable situation | Text post | Conversational, Danish, drives "tagging" | Medium |
| News reaction / landscape commentary | Text post | Timely, opinionated, brief | Low–Medium |
| Reusable reference / evergreen guide | Carousel (PDF) | High save rate, long tail via search | Very high |
